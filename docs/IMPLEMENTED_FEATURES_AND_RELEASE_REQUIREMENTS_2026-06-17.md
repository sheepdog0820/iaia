# 実装済み機能一覧と本番リリース要件

作成日: 2026-06-17

この文書は、現在のWebアプリ実装を本番リリース判断向けに要約したものです。詳細な機能説明は `docs/CURRENT_WEBAPP_FEATURES.md`、未完了課題は `ISSUES.md`、リリースリスク評価は `docs/test_results/PRODUCTION_READINESS_REVIEW_2026-06-17.md` を参照してください。

## リリース判断サマリー

現状は **クローズドベータまたは限定ユーザー向けリリースは条件付きで可能** です。

一般公開の本番リリースには、P0項目の完了に加えて、外部連携の実環境検証と障害時運用の確認が必要です。

## 本番リリースまでに必要な機能・作業

### P0: 一般公開前に必須

- 本番適用の最終承認
- Terraformの本番向け `plan` と `apply` の承認・実行
- 本番DB、ストレージ、ドメイン、TLS証明書、Secretsの確定
- Google、Discord、Twitter/Xなど外部サービスの本番OAuth設定
- 本番環境でのスモークテスト
- バックアップ取得とリストア手順の確認
- 監視、アラート、ログ確認手順の確定
- ロールバック手順の確認
- 管理者アカウントと初期運用権限の整理
- 利用規約、プライバシーポリシー、問い合わせ導線の準備

### P1: リリース前に強く推奨

- Google Calendar同期の実アカウント・実カレンダーでの失敗復旧検証
- Google Sheetsインポート/エクスポートの実シートでの失敗復旧検証
- Discord通知の失敗状態をユーザーまたは管理者が確認できる画面/API
- Discord通知の再送または運用復旧手順
- OpenAPIスキーマ警告の棚卸しと、公開APIに影響する警告の解消
- 主要画面のデスクトップ/モバイル表示確認
- 本番相当データ量での表示速度確認
- 外部連携ジョブの運用手順書化

## 現時点の主なリスク

- 本番インフラへの最終適用が未完了
- 外部連携は実装済みだが、実サービスを使った障害復旧検証が不足
- Discord通知は配信基盤がある一方、失敗の可視化と再送導線がGoogle連携より弱い
- APIスキーマ生成時に既存警告が残っている

## 推奨リリース方針

1. P0を完了するまで一般公開は行わない。
2. P0完了後、限定ユーザーでクローズドベータを開始する。
3. ベータ期間中にGoogle/Discord連携の失敗復旧を実地確認する。
4. 一般公開前に主要画面の表示品質と本番相当データ量での速度を確認する。
