# キャラクターバージョン作成方針

バージョン作成は `CharacterVersionService` のみが実行する。View、モデル互換メソッド、ロールバックはいずれもこのサービスを呼び出す。

原則として基本情報、版固有データ、背景、技能、装備、画像、秘匿HO、公開範囲、許可ユーザーをコピーする。画像はストレージ上の同一ファイルを参照する新しい関連レコードとして作成する。過去の技能成長履歴と期限付き共有リンクはコピーしない。

`share_token` は新しいシートの既定値により再発行される。クライアントが指定できるのは、バージョンメモ、コピー方針、および現在HP・MP・SANとメモだけである。所有者、親、番号、共有トークンなどのサーバー管理値は受け付けない。

サービスは単一トランザクションで系列のシートをロックし、系列内の最大番号の次を採番する。現行の親子構造では系列を表す列がないため、DBの複合ユニーク制約は将来のデータ構造変更時に追加する。
